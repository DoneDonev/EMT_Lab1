## Working solution without lombok support for EMT Lab1_B

Потребно е да се изработи API за изнајмување на сместувања со неколку функционалности како:

додавање ново сместување коешто може да се изнајми;
ажурирање на постоечки запис за одредено сместување;
бришење сместување кое веќе не е во добра состојба и не може да се изнајмува; и
обележување сместување како изнајмено.
За оваа лабораториска вежба потребно е да креирате соодветна Spring Boot апликација, односно API. Во однос на базата на податоци, потребно е да користите PostgreSQL во комбинација со Flyway за управување со миграциите.

Во рамки на апликацијата се чуваат следните податоци за сместувањата: id (Long), createdAt (LocalDateTime), updatedAt (LocalDateTime), name (String), category (enum), host (Host) и numRooms (Integer).
Категоријата на сместувањето може да биде: ROOM, HOUSE, FLAT, APARTMENT, HOTEL или MOTEL, а состојбата: GOOD или BAD.
За секој домаќин пак се чуваат податоците: id (Long), createdAt (LocalDateTime), updatedAt (LocalDateTime), name (String), surname (String) и country (Country).
За секоја земја се чуваат податоците: id (Long), name (String) и continent (String).
Користејќи слоевита архитектура треба да се дефинираат соодветни модели, репозиториуми, сервиси и контролери. За секој од ентитетите потребно е да внесете иницијални податоци преку миграција. За DTOs потребно е да дефинирате валидација таму каде што сметате дека е соодветно. За справување со грешките што можат да настанат при извршување дефинирајте соодветен exception handler. Исто така, со помош на SwaggerUI потребно е да прикажете успешно извршување на API повиците. Забелешка: Секаде каде што е проценувате дека е потребно може да ја надградите апликацијата со елементи што не се експлицитно спомнати погоре, а се покриени во материјалите од аудиториски вежби.
